# วิวัฒนาการสถาปัตยกรรม Heterogeneous Late Fusion เพื่อการระบุตัวตนผ่านพฤติกรรมการพิมพ์ภาษาไทย (Thai-Stylometry Identity)

## 1. บทนำ: นิยามของอัตลักษณ์ดิจิทัลผ่านพฤติกรรมการพิมพ์ (Introduction: Defining Digital Identity via Typing Behavior)

ในมิติของนิติมานุษยวิทยาคอมพิวเตอร์ (Forensic Computational Linguistics) พฤติกรรมการพิมพ์ไม่ได้เป็นเพียงการส่งผ่านข้อความ แต่คือ **"ลายนิ้วมือทางความคิด (Cognitive Fingerprint)"** ที่สะท้อนตัวตนในระดับจิตใต้สำนึก สไตโลเมทรี (Stylometry) หรือการวิเคราะห์ลักษณะการเขียน จึงกลายเป็นยุทธศาสตร์สำคัญในการสร้างระบบยืนยันตัวตนที่มีความปลอดภัยสูง เนื่องจากรูปแบบการเลือกใช้คำ (Lexical Choice) การลากเสียงสระ (Vowel Elongation) และจังหวะการเว้นวรรค เป็นพฤติกรรมที่ลอกเลียนแบบได้ยากกว่ารหัสผ่านแบบดั้งเดิม

โครงการ **Thai-Stylometry Identity** มุ่งเป้าในการเปลี่ยนผ่านจากการจำแนกข้อมูลเชิงสถิติเบื้องต้น ไปสู่ระบบการยืนยันตัวตนระดับ Biometric โดยต้องฝ่าฟันอุปสรรคเชิงโครงสร้างของภาษาไทยในบริบท Short-text ดังนี้:

- **Information Sparsity (ความเบาบางของข้อมูล):** ข้อความแชทมักสั้นและขาดโครงสร้างทางไวยากรณ์ที่สมบูรณ์ ทำให้การสกัดฟีเจอร์ทำได้ยาก
- **Broken Encoding (ปัญหาการถอดรหัส):** ข้อมูลดิบจากแพลตฟอร์ม Social Media มักอยู่ในรูป Latin-1 ซึ่งต้องผ่านกระบวนการ Forensic Decoding กลับเป็น UTF-8 เพื่อกู้คืนอัตลักษณ์ภาษาไทย
- **Imbalanced Data (ความไม่สมดุลของข้อมูล):** ปริมาณข้อมูลระหว่างเจ้าของบัญชี (Owner) และผู้แอบอ้าง (Impostor) มักไม่สมดุล จึงต้องใช้เทคนิค Downsampling เพื่อรักษาสมดุล 50:50 ในการสร้าง Decision Boundary ที่เป็นกลาง

การเดินทางของงานวิจัยนี้เริ่มจากการวิเคราะห์ Baseline เพื่อพิสูจน์ว่าเหตุใดวิธีการ Deep Learning แบบดั้งเดิมจึงล้มเหลวในโจทย์ด้านพฤติกรรม

## 2. ข้อจำกัดของโมเดลพื้นฐานและการพิสูจน์ความล้มเหลวของ Deep Semantic (Baselines and the Failure of Semantic-Heavy Models)

การสร้างบรรทัดฐาน (Baseline) คือหัวใจของการพิสูจน์ความก้าวหน้าทางสถาปัตยกรรม ในระยะแรกเราได้ทดสอบโมเดล Machine Learning แบบคลาสสิก ร่วมกับเทคนิค TF-IDF (Term Frequency-Inverse Document Frequency) ทั้งในระดับคำและ Character N-grams

### ตารางเปรียบเทียบประสิทธิภาพโมเดลพื้นฐาน (Baseline Models)

|   |   |   |   |
|---|---|---|---|
|Model|Accuracy|FAR (False Acceptance Rate)|FRR (False Rejection Rate)|
|Logistic Regression|75.87%|24.33%|**23.94%**|
|Linear SVM|75.78%|24.16%|24.28%|
|Random Forest|74.63%|25.87%|24.87%|

จากการวิเคราะห์ พบว่าค่า FAR ที่สูงถึง 24% นั้นไม่ผ่านเกณฑ์ความปลอดภัยระดับองค์กร เนื่องด้วยความล้มเหลวของโมเดลในการแยกแยะ "สไตล์" ออกจาก "เนื้อหา"

### ปรากฏการณ์ "Semantics vs. Style" Paradox

เมื่อขยับสู่โมเดลระดับสูงอย่าง **WangchanBERTa** หรือโมเดลในกลุ่ม LLM (Typhoon/Llama) ประสิทธิภาพกลับลดลงอย่างมีนัยสำคัญ เนื่องจาก:

1. **Semantic Normalization:** BERT ถูกฝึกมาเพื่อเข้าใจ "ความหมาย" (Semantics) ทำให้คำว่า "กิน" และ "ทาน" ถูกมองว่าใกล้เคียงกัน แต่ในงาน Stylometry ความแตกต่างนี้คือ Biometric Signal ที่สำคัญที่สุด
2. **Grammar Nazi Bias:** LLM มีอคติเชิงไวยากรณ์ โดยมักตัดสินว่าการพิมพ์ผิด (Typos) หรือ "นิ้วเบียด" เป็นลักษณะของผู้แอบอ้าง ทั้งที่เป็นพฤติกรรมปกติของมนุษย์
3. **Overfitting to Slang:** ในการทดลอง Contrastive Prompting พบว่า LLM มัก "ใจง่าย" ยอมรับผู้แอบอ้างเพียงเพราะมีการใช้คำสแลงยอดฮิต (เช่น "แปก", "ตึงจัด") โดยไม่ได้วิเคราะห์โครงสร้างประโยคเชิงลึก (Implicit Syntax)

## 3. การก้าวข้ามขีดจำกัดด้วย Char-CNN และกลไก Attention (Breaking Bottlenecks with Char-CNN and Attention)

เพื่อแก้ปัญหา Information Sparsity และภาษาแชทที่ไม่เป็นทางการ เราได้เปลี่ยนกระบวนทัศน์มาใช้ **Character-level 1D-CNN** ซึ่งมองข้อความในมิติของ "Morphological N-grams" ทำให้สามารถดักจับ Fingerprint ของการสะกดผิดและการลากเสียงสระได้อย่างแม่นยำโดยไม่ต้องพึ่งพา Dictionary

ในส่วนของมิติเวลา เราเริ่มด้วยสมมติฐาน **Time-Decay** ที่ให้น้ำหนักความสำคัญลดหลั่นตามความใหม่ของข้อความ: `Weight = (0.15 * m_{t-2}) + (0.35 * m_{t-1}) + (0.50 * m_t)`

อย่างไรก็ตาม เมื่อนำ **Dynamic Attention Mechanism** มาประยุกต์ใช้ ผลการวิเคราะห์น้ำหนัก Softmax กลับเผยความจริงที่น่าสนใจว่าโมเดลให้ค่าน้ำหนักใกล้เคียง **1:1:1** ซึ่งเป็นการค้นพบ (Discovery) ว่าในภาษาแชทไทย "Conversational Flow" หรือความต่อเนื่องของบทสนทนามีความสำคัญเท่าเทียมกับข้อความปัจจุบัน

**Performance Leap:** สถาปัตยกรรม Char-CNN ร่วมกับ Attention สามารถยกระดับความแม่นยำขึ้นสู่ **93.22%** พร้อมค่า FRR ที่ลดลงเหลือเพียง 5.08% ซึ่งถือเป็นจุดเปลี่ยนสำคัญสู่ระดับการใช้งานจริง

--------------------------------------------------------------------------------

## 4. กลยุทธ์ MLOps และการสร้างข้อมูลสังเคราะห์ (MLOps Strategy & Synthetic Augmentation)

ในเชิง MLOps การประมวลผล Backpropagation เต็มรูปแบบทุกครั้งเป็นเรื่องสิ้นเปลือง เราจึงเลือกใช้แนวคิด **"Decoupling"** แยกส่วนสกัดคุณลักษณะ (Feature Extractor) ออกจากส่วนตัดสินใจ (Classifier):

- **The Body (Universal Extractor):** ใช้ CNN ที่ถูก Freeze น้ำหนักเพื่อสกัด Context Vector 128 มิติ
- **The Head (Dynamic Classifier):** ใช้ XGBoost ที่มีน้ำหนักเบาและอัปเดตได้รวดเร็ว (Swappability) เพื่อสร้างกฎการตัดสินใจที่เฉียบขาด

### การรับมือกับข้อจำกัดด้านความเป็นส่วนตัว (PDPA)

เนื่องด้วยประวัติการแชทเป็นข้อมูลอ่อนไหว เราจึงใช้ **Llama 3.1** ในการสร้าง **Synthetic Data** เพื่อทำ Data Augmentation โดยจำลอง Persona ที่แตกต่างกัน เช่น:

- **User_A (Strict Boss):** เน้นประโยคสั้น กระชับ ไม่มีคำสร้อย
- **User_B (Polite Junior):** ใช้คำลงท้าย "ครับ/ค่ะ" และอีโมจิจำนวนมาก
- **User_C (Slang Master):** ใช้คำศัพท์เฉพาะกลุ่มและไวยากรณ์ที่พังพินาศ

กระบวนการนี้ช่วยให้โมเดลมี Generalization ที่ดีขึ้นและลดปัญหา Overfitting เมื่อต้องเผชิญกับข้อมูลจริงที่เบาบาง โดยผลทดสอบแบบ Mixed Model สามารถดัน Accuracy ขึ้นไปถึง 80.43% ในช่วงเริ่มต้น

## 5. สุดยอดสถาปัตยกรรม: Heterogeneous Late Fusion (The SOTA: Heterogeneous Late Fusion Architecture)

สถาปัตยกรรมขั้นสูงสุดถูกออกแบบบนหลักการ **"Feature Orthogonality"** เพื่อทะลวงขีดจำกัดของข้อความสั้น โดยผสานฟีเจอร์จาก 3 โดเมนที่ไม่ซ้ำซ้อนกัน:

### รายละเอียดโครงสร้าง 3 Parallel Branches:

1. **Branch 1: Deep Structural (Char-CNN + Attention):** ใช้ระบบ **Temporal Context Expansion (Sliding Window W=5)** เพื่อสกัดฟีเจอร์ทางวากยสัมพันธ์ (Implicit Syntax) ขนาด 128 มิติ
2. **Branch 2: Behavioral Heuristics (Regex-based):** สกัดคุณลักษณะพฤติกรรม 5 มิติ (ความยาว, จำนวนการหัวเราะ, การลากเสียง, วรรคตอน, และการเว้นวรรค) เพื่อเป็นความรู้จากผู้เชี่ยวชาญมนุษย์
3. **Branch 3: Statistical Stacking (TF-IDF + Logistic Regression):** เพื่อป้องกันปัญหา **Curse of Dimensionality** เราไม่นำฟีเจอร์ TF-IDF 3,000 มิติมาใช้โดยตรง แต่ใช้ Logistic Regression เป็น **Level-0 Base Estimator** เพื่อบีบอัดข้อมูล Lexical ให้เหลือเพียงค่าความน่าจะเป็น (Probability Scalar) 1 มิติ

ฟีเจอร์รวม 134 มิตินี้ จะถูกส่งเข้าสู่ **XGBoost Meta-Classifier** ซึ่งผ่านการปรับจูนด้วย **Bayesian HPO (Optuna)** โดยเฉพาะการทำ Regularization ผ่าน `subsample` และ `colsample_bytree` เพื่อให้ได้ Decision Boundary ที่รัดกุมที่สุดใน Manifold ของเจ้าของบัญชี

## 6. การวิเคราะห์ประสิทธิภาพและมาตรฐานความปลอดภัยระดับ Biometric (Performance Analysis and Security Implications)

การประเมินความปลอดภัยในระบบ Biometric ต้องให้ความสำคัญกับค่า FAR (โอกาสที่โจรหลุดรอด) เป็นอันดับหนึ่ง

### สรุปผลลัพธ์ระดับ State-of-the-Art (SOTA)

|   |   |   |   |
|---|---|---|---|
|Architecture|Accuracy|FAR|FRR|
|Baseline (TF-IDF + Voting)|76.32%|22.91%|24.46%|
|Char-CNN + Attention|93.22%|8.49%|5.08%|
|**Heterogeneous Late Fusion (SOTA)**|**97.70%**|**2.24%**|**2.36%**|

**วิเคราะห์ผลลัพธ์เชิง Synergistic Effect:** ความสำเร็จของ SOTA เกิดจากการปิดจุดอ่อนระหว่างกัน: Branch 3 (Lexical) ทำหน้าที่เป็น "สมอเรือ" จับคำศัพท์เฉพาะตัว, Branch 1 (CNN) ตรวจสอบโครงสร้างประโยคแม้โจรจะเลียนคำศัพท์ได้, และ Branch 2 (Heuristics) ทำหน้าที่เตะผู้แอบอ้างที่มีนิสัยพื้นฐานต่างกันออกไปอย่างเฉียบขาด

## 7. บทสรุป: อนาคตของการยืนยันตัวตนด้วย Stylometry (Conclusion: The Future of Stylometric Authentication)

วิวัฒนาการจาก TF-IDF พื้นฐานสู่สถาปัตยกรรม **Heterogeneous Late Fusion** ที่มีประสิทธิภาพสูงถึง **97.70%** คือเครื่องพิสูจน์ว่าพฤติกรรมการพิมพ์ภาษาไทยสามารถใช้เป็น Biometric ลำดับที่สองที่มีความมั่นคงสูงได้จริง

หัวใจสำคัญคือการผสาน Representation Learning (Deep Learning) เข้ากับสถิติดั้งเดิมและ Domain Knowledge ของมนุษย์ ภายใต้โครงสร้าง MLOps ที่ยืดหยุ่น นี่คือรากฐานสำคัญสู่ระบบ **"Continuous Authentication"** ที่สามารถเฝ้าระวังและยืนยันตัวตนของผู้ใช้งานได้แบบ Real-time บนแพลตฟอร์มรับส่งข้อความ ยกระดับความปลอดภัยทางไซเบอร์ของไทยสู่มาตรฐานสากลอย่างแท้จริง
